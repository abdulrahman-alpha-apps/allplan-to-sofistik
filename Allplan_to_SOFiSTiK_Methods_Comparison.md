### 1. 🛠️ Method Overview

#### Method 1 — SOFiSTiK SSD IFC/SAF Connector (GUI)
The built-in import pipeline inside SOFiSTiK Structural Desktop (SSD). You export an IFC file from Allplan and import it directly into SOFiSTiK using the IFC/SAF Connector task. The software auto-maps materials, cross-sections, and geometry to SOFiSTiK's internal format (CADINP).

#### Method 2 — Design2Static Plugin (Allplan Plugin)
A third-party plugin that lives entirely inside Allplan. It reads your architectural/formwork model and intelligently derives a structural analysis model — ignoring non-bearing elements, correcting bearing axes, and exporting directly to SOFiSTiK (or other FEA software). No IFC file required.

#### Method 3 — IfcOpenShell + Python → CADINP/TEDDY
A fully code-driven approach. You export IFC from Allplan, then parse it using the open-source Python library `ifcopenshell`, extract structural elements programmatically, and generate raw CADINP `.dat` files for SOFiSTiK's TEDDY text editor. Gives you 100% control over the output.

### 2. ⚙️ Technical Workflow

#### Method 1 — SSD IFC Connector

```
Allplan
  └─→ Export: File → Export → IFC 4 / IFC 2x3
          └─→ Select view: "Structural Analysis"
SOFiSTiK SSD
  └─→ New Project → "Import of a structural model"
      └─→ Add Task: IFC/SAF Connector
          └─→ General tab: load .ifc file
          └─→ Materials tab: review/fix auto-mapping
          └─→ Cross-Sections tab: review/fix auto-mapping
          └─→ Click OK → Model imported
              └─→ Right-click task → Text Editor
                  └─→ View/edit auto-generated CADINP code
                      └─→ Run FEA analysis
```

#### Method 2 — Design2Static Plugin

```
Allplan (with Design2Static installed)
  └─→ Activate D2S panel
      └─→ Select architectural/formwork model
          └─→ D2S generates structural scheme:
              - Ignores non-bearing elements
              - Derives correct bearing axes
              - Assigns structural properties
          └─→ Direct export to SOFiSTiK (no IFC needed)
              └─→ SOFiSTiK receives clean analysis model
                  └─→ Run FEA analysis
```

#### Method 3 — IfcOpenShell + Code

```
Allplan
  └─→ Export: IFC 4 Structural Analysis View (.ifc)

Python Script (IfcOpenShell)
  └─→ ifcopenshell.open("model.ifc")
      └─→ Extract: IfcBeam, IfcColumn, IfcSlab, IfcWall
          └─→ Get geometry, materials, profiles
          └─→ Map to SOFiSTiK CADINP syntax
              └─→ Write: sofistik_model.dat

SOFiSTiK TEDDY
  └─→ Open sofistik_model.dat
      └─→ Edit / extend CADINP manually
          └─→ Run via SSD or TEDDY directly (Ctrl+F5)
              └─→ FEA analysis
```

### 3. 🎯 Result Quality

#### Method 1 — SSD IFC Connector
- **Geometry accuracy:** Good — standard elements (beams, columns, slabs) transfer well
- **Material mapping:** Automatic but imperfect — standard materials (concrete C30, steel S355) map correctly; custom Allplan materials require manual correction
- **Cross-section mapping:** Standard profiles (IPE, HEB, HEA, RHS) auto-recognized; non-standard sections need manual assignment
- **Loads:** ❌ NOT transferred via IFC — must be defined manually in SOFiSTiK after import
- **Connections/joints:** Basic only — complex node connections may need rework
- **Mesh quality:** Auto-generated by SOFiSTiK — adequate for standard structures
- **Overall rating:** ⭐⭐⭐ (3/5) — reliable for standard cases, cleanup needed for complex ones

#### Method 2 — Design2Static Plugin
- **Geometry accuracy:** Very good — model is derived from Allplan's internal data (not IFC), so data loss is minimal
- **Material mapping:** High — plugin has dedicated mappings per FEA software (SOFiSTiK-specific export)
- **Cross-section mapping:** High — structural scheme is interpreted, not just geometry-exported
- **Loads:** ⚠️ Partial — some load types can be included; depends on plugin version
- **Connections/joints:** Better than IFC — structural logic is preserved
- **Mesh quality:** Good — structural scheme is the basis, not raw geometry
- **Overall rating:** ⭐⭐⭐⭐ (4/5) — significantly cleaner model than raw IFC import

#### Method 3 — IfcOpenShell + Code
- **Geometry accuracy:** Excellent — you control exactly what is read and how it's written
- **Material mapping:** Excellent — you define your own mapping tables (e.g., `C30/37 → NMAT 1 TYPE C30`)
- **Cross-section mapping:** Excellent — full control; can handle non-standard custom profiles
- **Loads:** ✅ Can be fully included — if load data exists in IFC (IfcStructuralLoad*), you can extract and convert it
- **Connections/joints:** Excellent — write custom CADINP for any connection type
- **Mesh quality:** Excellent — define mesh density, element type, and topology yourself
- **Overall rating:** ⭐⭐⭐⭐⭐ (5/5) — best possible quality, but requires deep knowledge of both IFC and CADINP

### 6. 📚 Learning Curve

#### Method 1 — SSD IFC Connector
- **Required knowledge:** Basic SOFiSTiK SSD, basic IFC export in Allplan
- **Time to first working model:** ~1–2 hours (including reading docs)
- **Common pitfalls:**
  - Forgetting to select "Structural Analysis" view in Allplan IFC export → get architectural geometry instead
  - Materials not mapping correctly → structural analysis uses wrong properties
  - No loads transferred → must add all loads manually

#### Method 2 — Design2Static Plugin
- **Required knowledge:** Allplan modeling conventions, basic structural scheme concepts
- **Time to first working model:** ~4–8 hours (plugin setup + learning workflow)
- **Common pitfalls:**
  - Plugin must be set up for each Allplan project
  - Architectural model must be modeled "correctly" (proper layer/attribute discipline) for D2S to interpret it well
  - Complex architectural forms may not transfer cleanly

#### Method 3 — IfcOpenShell + Code
- **Required knowledge:** Python programming, IFC schema understanding (IfcBeam, IfcRelAssociatesMaterial, etc.), SOFiSTiK CADINP syntax, TEDDY/SSD workflow
- **Time to first working model:** ~3–7 days for a developer new to IFC
- **Time for an experienced developer:** ~1–2 days for a simple model
- **Common pitfalls:**
  - IFC geometry extraction is complex — coordinates, local placement, transformations
  - CADINP syntax errors crash the task silently
  - Material/profile name mismatches between IFC and SOFiSTiK database
  - IFC from Allplan may not always include structural properties (depends on export settings)
